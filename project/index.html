What do you mean "that's already been done"?

Feature rich

We have all the basics such as saving, loading, undo and redo covered. You can also visualise all possible moves and threats in case you are a learning player.

Colorschemes

Infinite diversity in infinite combinations. Switch between any combination of colors by using either the built-in colorschemes, or make your own using our simple templates.

Documented

All code, outward and inward facing, is documented using javadoc-style comments so that the document generation tool of your choice such as doxygen or Sphinx can easely generate a comprehensive manual.

diagram

This diagram is a high-level overview of the structure of the chess program. The meaning of the colors is explained in the bottom-right corner.

The main class has an instance of the Game object, which is the overarching class of the program. This class inherits from ChessWindow and includes the non-local type QApplication and the local abstract type ChessPiece. These parts contain the chess logic.

The UI is handled by ChessWindow and ChessScene. Most of the UI code is found in ChessScene. This is all done using inheritance and wrappers but there is one peculiarity: the click even listener. This listener can be found in ChessScene but this has to be responded to in the Game class, so in this case we work using a callback.
